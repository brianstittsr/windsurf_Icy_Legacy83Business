# SVP OEM Client Acquisition Funnel
## A Theory of Constraints Approach

*Based on principles from Theory of Constraints, The Choice, and Critical Chain by Eliyahu M. Goldratt*

---

## Executive Summary

This document applies Goldratt's Theory of Constraints (TOC) methodology to Strategic Value Plus's OEM client acquisition process. By identifying and exploiting the constraint in our sales funnel, we can dramatically improve throughput (closed deals) while reducing the time and resources required.

**Core TOC Principle Applied**: *"A chain is only as strong as its weakest link."*

The goal is not to optimize every stage of the funnel, but to identify THE constraint that limits our ability to convert OEM prospects into clients, and focus all efforts on exploiting and elevating that constraint.

## Part 1: The Thinking Process

### From "The Choice" - Clarity, Not Complexity

Goldratt's "The Choice" teaches us that:
1. **Reality is simple** - Complex problems have simple root causes
2. **Conflicts can be resolved** - Every conflict has a win-win solution
3. **People are good** - Resistance comes from unaddressed concerns, not malice
4. **Never say "I know"** - Always question assumptions

## Part 3: The Critical Chain Sales Process

### From "Critical Chain" - Buffer Management for Sales

Critical Chain teaches us that:
1. **Student Syndrome**: Work expands to fill available time
2. **Multi-tasking is the enemy**: Focus on one thing at a time
3. **Buffers protect the critical path**: Build in safety, but manage it centrally
4. **Relay race mentality**: Pass the baton as fast as possible

#### The Critical Chain of OEM Acquisition

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   CRITICAL CHAIN: OEM CLIENT ACQUISITION                                    │
│   ══════════════════════════════════════                                    │
│                                                                             │
│   Traditional Approach (with hidden safety in each step):                   │
│   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐                │
│   │ 30d │ → │ 14d │ → │ 21d │ → │ 14d │ → │ 21d │ → │ 45d │ = 145 days    │
│   │Lead │   │ MQL │   │ SQL │   │Disc │   │Prop │   │Close│                │
│   └─────┘   └─────┘   └─────┘   └─────┘   └─────┘   └─────┘                │
│                                                                             │
│   Critical Chain Approach (aggressive estimates + project buffer):          │
│   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────────┐  │
│   │ 15d │ → │  7d │ → │ 10d │ → │  7d │ → │ 10d │ → │ 21d │ → │ BUFFER  │  │
│   │Lead │   │ MQL │   │ SQL │   │Disc │   │Prop │   │Close│   │  30 days│  │

```

## Part 10: Key Takeaways

### The TOC Mindset for Sales

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   REMEMBER THESE PRINCIPLES                                                 │
│   ═════════════════════════                                                 │
│                                                                             │
│   From "Theory of Constraints":                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │ 1. There is always ONE constraint limiting throughput               │  │
│   │ 2. Optimizing non-constraints is waste                              │  │
│   │ 3. The constraint determines the pace of the entire system          │  │
│   │ 4. Exploit before you elevate                                       │  │
│   │ 5. When the constraint moves, start over                            │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   From "The Choice":                                                        │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │ 1. Reality is simple - find the root cause                          │  │
│   │ 2. Every conflict has a win-win solution                            │  │
│   │ 3. People resist because of unaddressed concerns                    │  │
│   │ 4. Never say "I know" - always question assumptions                 │  │
│   │ 5. Use thinking processes to find clarity                           │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   From "Critical Chain":                                                    │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │ 1. Remove hidden safety from individual steps                       │  │
│   │ 2. Create a project buffer to protect the end date                  │  │
│   │ 3. Use buffer status (Green/Yellow/Red) for management              │  │
│   │ 4. Relay race mentality - pass the baton fast                       │  │
│   │ 5. Multi-tasking destroys throughput                                │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   THE ULTIMATE GOAL:                                                        │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   "Make more money now AND in the future"                           │  │
│   │                                                                     │  │
│   │   For SVP, this means:                                              │  │
│   │   • Close more OEM deals (throughput)                               │  │
│   │   • Close them faster (cycle time)                                  │  │
│   │   • With less effort (operating expense)                            │  │
│   │   • While building long-term relationships (future throughput)      │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
